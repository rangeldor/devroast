# Specs Guidelines

## Formato de Spec

Cada spec deve seguir esta estrutura:

```
# [Nome da Feature]

## Objetivo
Breve descrição do que a feature faz e seu propósito.

---

## Requisitos
Lista de funcionalidades required (checkbox marker).

---

## Opções Pesquisadas
Análisis de 2-4 alternativas com:
- Nome da ferramenta/biblioteca
- Prós e contras
- Bundle size se aplicável
- Referências

---

## Recomendação
Justificativa da escolha final.

---

## Implementação Proposta

### Stack
Tecnologias/bibliotecas escolhidas.

### Estrutura
Estrutura de arquivos sugerida.

### Features
Lista de items com [x] para implementado e [ ] para pendente.

---

## To-Dos
Lista de tarefas para implementar a feature.
```

## Regras

- Use `---` para separar seções principais
- Sempre inclua prós/contros nas opções
- Documente o bundle size se relevante
- Adicione referências (links, artigos)
- Mantenha conciso - sem redundância
